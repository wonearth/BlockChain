# 3장 포크와 알트체인

## 비트코인 개선안

- 비트코인 개선안(BIP, Bitcoin Improvement Proposal)은 비트코인 프로토콜의 변경사항을 제안하는 문서
  - 소프트웨어 제품의 일반적인 개선 방식
  - BIP는 비트코인 커뮤니티에서 논의되며, 커뮤니티에서 동의해서 채택되면 비트코인 프로토콜에 반영될 수 있음
  - 모든 BIP는 Github에 공개되어 있으며, 누구나 제안할 수 있음

### 포크

- 비트코인 커뮤니티가 네트워크의 공동 이익을 위해 함께 모이기 시작함
  -> 점점 더 많은 프로그래머들이 소프트웨어 포크를 통해 개량한 비트코인을 만들기 시작함
  -> altcoin(대체코인)이라는 새로운 암호화폐가 탄생함
- 포크는 크게 하드포크와 소프트포크로 나뉨

| 소프트포크                                 | 하드포크                            |
| ------------------------------------------ | ----------------------------------- |
| 기존 블록체인과 호환(일부 매개변수만 변경) | 기존 블록체인과 호환되지 않음       |
| 네트워크 변경O >> 모든 채굴자 참여X        | 네트워크 변경X >> 모든 채굴자 참여O |

</br>

### 하드포크

- 일부 마이너들이 포크에 동의하지 않아, 기존 블록체인과 호환되지 않는 새로운 블록체인이 만들어지는 경우
  -> 새로운 소프트웨어 버전으로 업그레이드하지 못해 블록체인이 사실상 둘로 갈라짐

### 마이너

- 네트워크 실행을 위해, 해시파워를 기여하는 사람들
- 블록체인 네트워크에서 거래를 검증하고 & 블록을 생성하는 역할을 하는 사람 또는 조직
  - 해시파워 >> 작업 증명 기반 블록체인의 생존에 중요함
  - 거래 검증: 마이너는 거래가 유효한지 확인하고, 거래가 블록체인에 추가될 수 있도록 승인함
  - 블록 생성: 마이너는 거래를 모아서 블록을 만들고, 이를 블록체인에 추가함
  - 보상: 마이너는 새로운 블록을 생성할 때마다 일정량의 암호화폐로 보상을 받음

</br>

### 리플레이 공격 (하드포크 발생 시)

- 공격자가 한 블록체인에서 합법적인 거래의 데이터를 가져다가 두 번쨰의 블록체인에서 해당 거래를 복제 or 미러링할 때 발생
- 예시: A가 B에게 1비트코인을 보내는 거래가 블록체인에서 발생했다고 가정
  -> 공격자는 이 거래의 데이터를 복제하여, 두 번째 블록체인에서 A가 B에게 1비트코인을 보내는 거래로 만들어서 실행할 수 있음
  -> 이로 인해 A는 의도치 않게 두 번째 블록체인에서도 1비트코인을 보내게 될 수 있음

</br>

- 비트코인캐시 포크가 발생했을 때 새로운 소프트웨어 비트코인캐시 마이너는 어느 블록체인에서든 리플레이 공격이 일어나지 않도록 변경사항을 포함

</br>

### 비트코인캐시 포크

- 2017년 8월 1일, 비트코인캐시라는 새로운 블록체인이 비트코인에서 하드포크로 만들어짐
- 비트코인캐시는 블록 크기를 1MB에서 8MB로
  -> 거래 처리량을 늘리고, 거래 수수료를 낮추는 것을 목표로 함
  - 비트코인캐시는 여전히 가장 성공적인 **비트코인 소프트웨어 포크**로 간주됨

</br>

### 알트코인

- 비트코인 코어 소프트웨어의 포크를 가리킬 때 쓰는 용어
- 아이엑스코인, 라이트코인, 대시, 지캐시 등 다양한 알트코인이 존재

</br>

### 라이트코인

- 초기부터 잘 알려진 알트코인
  -> 비트코인을 금과 비슷한 가치의 저장소로 생각해, 비트코인을 보완한 코인을 만들었다.
- 라이트코인은 비트코인보다 빠른 거래 확인 시간과 더 낮은 수수료를 제공하는 것을 목표로 함

</br>

### '2.0' 체인

| 넥스트(NXT)                    | 카운터파티                                       |
| ------------------------------ | ------------------------------------------------ |
| 프로그램적이고 유연한 블록체인 | 마스터코인처럼 비트코인의 블록체인 확장 프로토콜 |
| 실제로 도입된 기능이 거의 없음 | 비트코인 블록체인 위에 구축된 플랫폼             |
| ------------------------------ | XCP라고 하는 자체 암호화폐를 갖고 있다           |

</br>

### 개인정보 보호에 중점을 둔 암호화폐

- 대시(Dash)와 모네로(Monero) 프라이버시를 강화하는 암호화폐
  -> 거래의 발신자, 수신자, 금액 등의 정보를 암호화하여 보호함
  -> 이러한 암호화폐는 개인 정보 보호에 민감한 사용자들에게 인기가 있음

| 대시(Dash)                     | 모네로(Monero)                                                                   | 지캐시(Zcash)                                                         |
| ------------------------------ | -------------------------------------------------------------------------------- | --------------------------------------------------------------------- |
| 비트코인의 소프트웨어 포크     | 작업 증명을 위해 크립토노트CryptoNote 프로토콜을 사용                            | 오늘날 가장 잘 알려진 개인정보 보호 체인                              |
| 실제로 도입된 기능이 거의 없음 | 링 시그니처 (키를 소유한 집단이 정보를 숨길 목적으로 사용할 수 있는 디지털 서명) | 사용자의 개인정보를 보호하기 위해 영 지식 증명(zk-SNARKs) 기술을 사용 |
| ASIC 저항 작업 증명 구현       | XCP라고 하는 자체 암호화폐를 갖고 있다                                           | 실제 데이터를 공유하지 않고도 서로 정보를 전송 가능                   |

</br>

### 리플과 스텔라

- 배경: 소프트웨어 포크로 비트코인 기반의 작업 증명 합의 알고리즘 적용 암호화폐 ??
  -> 점점 *중앙화*된 시스템이 됨

- 리플과 스텔라는 중앙화된 시스템이지만, 빠른 거래 처리와 낮은 수수료를 제공하는 것 + 증명 방식을 사용하지 않는 암호화폐로 설계됨
- 리플은 금융 기관 간의 국제 송금을 위한 솔루션으로 설계되었으며, 스텔라는 개인 간의 거래를 촉진하는 것을 목표로 함

</br>

## 블록체인

- 중앙화된 VISA와 달리 비트코인은 탈중앙화로 네트워크가 구성돼 있음 (중앙 제어 X)
- 블록체인은 거래의 투명성과 보안을 제공하며, 중앙 기관 없이도 거래를 검증할 수 있도록 함

</br>

## 이더리움 클래식 포크

- 이더리움에서 하드포크가 발생하여 이더리움 클래식이 만들어짐
- 이더리움 클래식은 이더리움의 *원래 블록체인을 유지*하면서, 이더리움의 하드포크로 인해 발생한 거래를 되돌리는 것을 거부
- 이더리움 클래식은 이더리움과는 별개의 블록체인으로 존재하며, 자체적으로 거래를 처리하고 스마트 계약을 실행
